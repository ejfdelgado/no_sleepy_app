Include, authentication with firebase with: 1. google account, 2. cellphone or 3.celphone number.

A second view allows to add or edit a location alarm, on top one text field allows to name the alarm, a second text field allows to write and search an address with a button at rigth to search and display the location in a map below, also in the map the user could relocate the latitud and longitude manually dragging the map, then, below there is a drop down to pick a sound alarm among five different sounds. Store all in firestore, using the following collections: 1. "alarm_user" collection for registered users usin the user uid as the id, 2. "alarm_item" to store the alarm item to persist, name, writed address, latitude, longitude, created, updated date, sound picked, the user uid owner, and an autogenerated id, also include a collection "alarm_advertise", to store a title, description, a url image with a latitud and longitude in order to display to the user the closest business when it get close to the current gps location. Allows to define in the parameter project: 1. the google maps api key to allow search latitude and longitud by typed address, 2. a number defining the distance threshold to fire alarm or closest business, 3. a timeout number in millis to defone how offen the app recompute uf it needs to fire some alarm.
